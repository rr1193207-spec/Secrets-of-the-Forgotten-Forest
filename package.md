# Пакет разработки игры

## 1. Краткий бриф

**Название:** Тайны Забытого Леса

**Жанр и камера:** 2D-платформер, вид сверху

**Одно предложение:** Исследуйте тёмный лес, собирайте ключ, открывайте врата и активируйте древний артефакт, пока не погас фонарь

**Платформа:** мобильная

**Экран:** 16:9

**Управление:** touch

**Игровая цель:**
Найти ключ, открыть врата, активировать артефакт

**Сеттинг и тон:**
Мрачный лес в сумерках, атмосфера тайны, приглушённые цвета

## 2. MVP

**Гипотеза:**
Игрок сможет пройти локацию за 3–5 минут, управляя светом фонаря и избегая ловушек

**Входит:**
Локация 6×5 тайлов

Герой с фонарём

Ключ и врата

Артефакт

Шкалы HP и фонаря

Ловушки

**Не входит:**
Дополнительные уровни

Система сохранений

Прокачка

Диалоги

**Критерий успеха:**
Игрок активирует артефакт, не потеряв все HP и заряд фонаря

## 5. Механики и состояния

**Основной цикл:**
Сбор ключа --} Открытие врат --} Активация артефакта

| Механика | Событие | Условие | Что меняется | Результат для игрока |
| --- | --- | --- | --- | --- |
| Сбор ключа | Игрок подходит к ключу и нажимает E / тап | Расстояние до ключа ≤ 1 тайла; ключ ещё не собран | Флаг hasKey = true; ключ исчезает из мира; проигрывается звук подбора | На экране появляется подсказка «Ключ получен» |
| Открытие врат | Игрок подходит к вратам и нажимает E / тап | Расстояние до врат ≤ 1 тайла; hasKey = true | Флаг gatesOpen = true; врата анимируются и открываются; проигрывается звук щелчка | Врата визуально открываются, мини‑цель обновляется на «Активируй артефакт»; открывается путь к артефакту |
| Активация артефакта | Игрок подходит к вратам и нажимает E / тап | Расстояние до артефакта ≤ 1 тайла; gatesOpen = true | Флаг artifactActive = true; запускается триггер победы | На экране появляется эффект свечения артефакта, запускается сцена победы (переход в gameState = win) |
| Расход заряда фонаря | Каждую секунду игрового времени | Игра в состоянии play; заряд фонаря > 0 | Переменная lanternCharge -= 1 | Шкала заряда на UI уменьшается; при низком заряде появляется предупреждение «Фонарь гаснет» |

**Главная механика:**
Механика расхода заряда фонаря и визуального круга света

**gameState:**
menu,play,pause,win,lose

## 7. Данные, псевдокод и код

| Переменная | Тип | Начальное значение | Когда меняется | Зачем нужна |
| --- | --- | --- | --- | --- |
| game_state | строка | menu | при переходах | разрешает действия только в нужном состоянии |
| energy | число | 0 | при сборе и трате | ограничивает взаимодействия |
| hp | число | 3 | при уроне и лечении | условие поражения |
| max_hp | число | 3 | редко меняется | верхняя граница HP |
| score | число | 0 | при успешных действиях | показывает прогресс |
| object_used | флаг | False | после взаимодействия | не даёт выполнить действие повторно |
| portal_open | флаг | False | после выполнения условий | разрешает победу или переход |
| is_paused | флаг | False | при паузе | останавливает обновления |

### Псевдокод

```text
если game_state == 'play':
   если игрок коснулся врат:
      если ключей > 0:
         открыть врата
      иначе:
         показать ошибку("Недостаточно ресурса!")
```

**Функции-обёртки:**


**Язык:** Python

**Где проверяли:** браузере

```python
if game_state != 'play':
    return

if player.rect.colliderect(object_rect):
    if resource > 0:
        perform_action(resource)
    else:
        show_message("Недостаточно ресурса!")
```

## 8. Тесты и риски

| Тест | Входные данные | Действие | Ожидаемый результат | Факт | Статус |
| --- | --- | --- | --- | --- | --- |
| Успешное действие | energy = 1, object_used = False | Игрок касается объекта | Объект активирован, energy уменьшается |  |  |
| Нет ресурса | energy = 0 | Игрок пытается активировать объект | Появляется сообщение, состояние не меняется |  |  |
| Повторное действие | object_used = True | Игрок касается объекта снова | Ресурс не тратится, счётчик не растёт |  |  |
| Объект уже использован | object_used = True | Игрок касается объекта снова | Состояние не меняется, ресурс не тратится |  |  |
| Ресурс на нуле при повторном касании | energy = 0, object_used = False | Игрок касается объекта снова | Состояние не меняется, появляется сообщение «Недостаточно энергии» |  |  |
| Выход из зоны взаимодействия | energy = 1, object_used = False | Игрок отходит от объекта (расстояние > 1 тайла) | Объект не активируется, состояние не меняется |  |  |

| Критическая точка | Что может сломаться | Защита в алгоритме | Как проверить |
| --- | --- | --- | --- |
| energy = 0 | ресурс может уйти в минус | проверять energy > 0 до траты | тест Нет ресурса |
| object_used = True | счётчик сработает повторно | проверять флаг использования | тест Повторное действие |
| hp = 0 | игрок продолжит играть | сразу менять game_state на lose | тест Урон при HP=1 |
| pause | механики продолжают работать | проверять game_state == play | тест Пауза |
| portal_open = True | открытие повторяется | проверять состояние портала | тест Повторный вход |

**Главный риск:**
ложное срабатывание коллизий из‑за несовпадения хитбоксов с тайловой сеткой, из‑за чего игрок сможет активировать объекты сквозь стены или не сможет подобрать ресурс на границе тайла.

## 9. План реализации

План реализации показывает порядок сборки прототипа, ориентиры по срокам и критерии готовности. Это помогает не уходить в визуальные детали до того, как заработала главная механика.

| Этап | Что сделать | Ориентир по сроку / дедлайн | Готово, когда... |
| --- | --- | --- | --- |
| 1. Состояния игры | Создать menu / play / pause / win / lose | 20–30 минут или до следующего занятия | Переходы работают без механик |
| 2. Управление | Добавить движение героя и границы уровня | 30–45 минут | Игрок двигается и не ломает сцену |
| 3. Главная механика | Собрать цепочку событие → условие → изменение | 1 занятие | Есть успешный и ошибочный сценарий |
| 4. Победа и поражение | Добавить условия win / lose и перезапуск | 30–45 минут | Игра корректно завершается |
| 5. Минимальный визуал | Подключить фон, героя, UI и ключевые объекты | после готовой логики | Все важные элементы читаемы |
